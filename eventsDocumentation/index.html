Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listened on the admin end to identify all available normal users that are currently online.
The output will be an object, object has key as userId and value as full name.
meetings
This event ("meetings") has to be listened at normal user side to get meetings which is created by admin.
Example output it as follows: 
 {
meetingId":Meeting Id,
"meetingCreatorId":Admin Id who is creating meeting,
"meetingCreatorName":Admin name who is creating meeting,
"title":Meeting title,
"start":start time of meeting,
"end":end time of meeting,
"venueName":Meeting venue,
"meetingPurpose":Purpose,
"meetingPartnerId": User id with whom meeting is scheduled,
"meetingPartner":User name with whom meeting is scheduled,
"createdOn":creation date,
"modifiedOn":modification date
}

Events to emit

setUser event
This event ("setUser") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token. The following data has to be emitted:
authentication token.
create-meeting
This event ("create-meeting") has to be emitted when a admin wants to create a meeting.
The following data has to be emitted for creating meeting:
{
meetingId":Meeting Id,
"meetingCreatorId":Admin Id who is creating meeting,
"meetingCreatorName":Admin name who is creating meeting,
"title":Meeting title,
"start":start time of meeting,
"end":end time of meeting,
"venueName":Meeting venue,
"meetingPurpose":Purpose,
"meetingPartnerId": User id with whom meeting is scheduled,
"meetingPartner":User name with whom meeting is scheduled,
"createdOn":creation date,
"modifiedOn":modification date
}
update-meeting
This event ("update-meeting") has to be emitted when a admin wants to update meeting.
The following data has to be emitted:
{
meetingId":Meeting Id,
"meetingCreatorId":Admin Id who is creating meeting,
"meetingCreatorName":Admin name who is creating meeting,
"title":Meeting title,
"start":start time of meeting,
"end":end time of meeting,
"venueName":Meeting venue,
"meetingPurpose":Purpose,
"meetingPartnerId": User id with whom meeting is scheduled,
"meetingPartner":User name with whom meeting is scheduled,

}
delete-meeting
This event ("delete-meeting") has to be emitted, when the admin wants to delete a meeting.
The following data has to be emitted
{
meetingId":Meeting Id,
"meetingCreatorId":Admin Id who is creating meeting,
"meetingCreatorName":Admin name who is creating meeting,
"title":Meeting title,
"meetingPartnerId": User id with whom meeting is scheduled,
"meetingPartner":User name with whom meeting is scheduled,

}
remind-event
This event ("remind-event") has to be emitted, when the meeting is going to start within 15 minutes for mailing to the concerned user.
The following data has to be emitted
{
meetingId":Meeting Id,
"meetingCreatorId":Admin Id who is creating meeting,
"meetingCreatorName":Admin name who is creating meeting,
"title":Meeting title,
"start":start time of meeting,
"end":end time of meeting,
"venueName":Meeting venue,
"meetingPurpose":Purpose,
"meetingPartnerId": User id with whom meeting is scheduled,
"meetingPartner":User name with whom meeting is scheduled,

}
remind-event-update
This event ("remind-event-update") has to be emitted to update the status of snooze to dismiss.
The following data has to be emitted
{
"meetingId":meetingId,
"user":type of user (normal or admin)
  }
mail-remind-event-update
This event ("mail-remind-event-update") has to be emitted to update the mail sent status to 1 (mail is sent).
The data has to be emitted
{
"meetingId":meetingId,
"user":type of user (normal or admin)
  }
disconnect
This event ("disconnect") has to be emitted, when the user gets disconnected(logout).
